## Purpose

Process 14 covers social-profile backlink and entity-link work for Lawn & Land SEO clients. The goal is to make sure the client's active social profiles are accurate, live, linked from the right places, and reinforce the same business identity used on the website, GBP, citations, and directory listings.

This is not a social media posting process. It is a profile/link hygiene and backlink process.

The goal is simple:

- Collect approved client social profile URLs.
- Verify each profile is real, active, and belongs to the correct business.
- Add only active, working social profile links where the workflow calls for them.
- Confirm social profiles link back to the correct client website when possible.
- Avoid linking to wrong, old, duplicate, inactive, or unrelated profiles.
- Route profile creation, claiming, login, and verification work to a human when needed.
- Document every live profile URL, missing profile, blocker, and follow-up.

## Target State

Every active SEO client should have a verified social profile inventory.

The ideal system:

1. Collects social profile links from approved client documents and known client assets.
2. Verifies each profile belongs to the correct client.
3. Rejects inactive, duplicate, wrong-brand, wrong-location, or personal profiles.
4. Confirms profile links on the website, GBP, directories, and tracking records are accurate.
5. Confirms social profiles link back to the correct client website where the platform allows it.
6. Captures missing or inaccessible profiles with a clear human owner.
7. Routes account creation, claiming, and credential work through approved access workflows.
8. Keeps social/entity links aligned with citation and backlink records.

## Profile Types

Prioritize active profiles that matter for entity trust, client visibility, or profile completeness.

| Profile Type | Examples | Rule |
|---|---|---|
| Core social profiles | Facebook, Instagram, LinkedIn, YouTube | Verify first when client provides links or profiles already exist. |
| Local/community profiles | Nextdoor, local community pages | Use when relevant and official. |
| Visual/service proof profiles | Instagram, YouTube, Pinterest | Useful when profile is active and matches the client brand. |
| Directory/social hybrids | Facebook profile inside directory/citation workflows | Verify NAP and website link as entity signals. |
| Personal profiles | Owner personal Facebook/LinkedIn | Do not use unless explicitly approved as the business profile. |
| Inactive or abandoned profiles | Old, empty, duplicate, or wrong-brand pages | Do not add without human review. |

## Workflow

### 1. Confirm Scope

Start from the ClickUp task `Social Profile`.

Confirm:

- The request is for verified social profile links, not social posting.
- The client is active and in scope for SEO/profile hygiene work.
- The profiles will be used for entity consistency, website links, GBP/profile records, directory records, or backlink tracking.
- Any login, claim, or ownership task has a human owner.

Do not create, claim, or edit social accounts with personal, generic, or unauthorized credentials.

### 2. Collect Candidate Profile Links

Collect social links from:

- Client onboarding documents.
- Client website header, footer, contact page, and schema.
- GBP public profile where visible.
- Existing citation or directory records.
- Known social profile inventories.
- Search results for the client name plus platform.
- Prior team notes or brand asset folders.

For each candidate, capture:

- Platform.
- Profile URL.
- Source of the URL.
- Business name shown.
- Website link shown.
- Phone/address if visible.
- Last activity or visible freshness when available.
- Whether login access is known.
- Initial decision: verified, needs review, wrong profile, duplicate, inactive, or blocked.

### 3. Verify Profile Ownership and Fit

For each profile, verify:

- The profile name matches the approved client name or accepted DBA.
- The website link points to the approved client website.
- The phone/address/service-area data does not conflict with the approved NAP.
- Branding is recognizable as the client.
- Photos, services, markets, or posts are not clearly for another business.
- The profile is live and accessible.
- The profile is not a personal page unless explicitly approved.
- There is not a better official business profile for the same platform.

If ownership is unclear, mark the profile as `needs human review` instead of adding it.

### 4. Check Website Social Links

Review the website locations where social links commonly appear:

- Header.
- Footer.
- Contact page.
- About page.
- Sidebar widgets.
- Schema/social sameAs fields.
- Page builder global settings.
- Theme customizer settings.

For each existing link:

- Confirm it resolves.
- Confirm it points to the official client profile.
- Confirm it opens the correct platform.
- Remove or flag broken, wrong, duplicate, or placeholder links.
- Confirm icon labels and ARIA text are reasonable when editable.

Do not add inactive profiles just to fill icon slots.

### 5. Check Social Profiles for Website Backlinks

Where public profile fields allow it, verify:

- Website URL points to the canonical domain.
- Link is not missing.
- Link does not point to an old domain, staging site, Linktree, unrelated booking tool, or wrong client.
- Profile description does not conflict with the approved service list or service area.
- Phone/address fields do not conflict with the approved NAP.

Some platforms hide website links or require login to edit. If a fix requires login, assign it to a human and document the blocker.

### 6. Decide What to Add, Fix, or Reject

Use a consistent classification.

| Status | Meaning |
|---|---|
| Verified | Correct official profile, safe to add/use. |
| Already linked | Website/profile records already use the correct link. |
| Needs update | Official profile exists but link/NAP/website field needs correction. |
| Missing | No official profile found or supplied. |
| Needs access | Fix requires login/admin access. |
| Duplicate | Multiple profiles exist and human review is required. |
| Wrong profile | Profile belongs to another business/location/person. |
| Inactive | Profile appears abandoned or unsuitable. |
| Blocked | Cannot proceed without client/platform/team action. |

### 7. Prepare the Update Packet

For each approved profile, prepare:

- Platform.
- Final URL.
- Where it should be added.
- Current website link status.
- NAP consistency notes.
- Whether login is required.
- Recommended update.
- Owner.
- Evidence URL or screenshot note.

For missing or blocked profiles, document the needed human action instead of making assumptions.

### 8. Apply or Assign Updates

Koga can prepare updates and verify public links.

Human action is required for:

- Creating a new social profile.
- Claiming or merging pages.
- Editing profile fields behind login.
- Meta Business Suite access.
- Phone/email verification.
- Ownership disputes.
- Removing duplicate pages.
- Any profile that uses sensitive business-owner identity.

If direct site edits are authorized and safe, update website social links only after the profile is verified. Otherwise prepare exact instructions for the site editor.

### 9. Verify Final Output

After updates, verify:

- Website social icons/links open the right profiles.
- Links do not redirect to wrong accounts.
- Social profile website fields point back to the correct site where visible.
- NAP and brand identity are consistent.
- No placeholder links remain.
- Any blocked or access-required item has an owner and next action.
- Final social inventory is recorded.

## Completion Criteria

The process is complete when:

- Official social profile links are collected and verified.
- Website social links are correct or update instructions are ready.
- Public social profiles point back to the correct website where visible.
- Wrong, duplicate, inactive, missing, or access-required profiles are documented.
- Any human-owned access or ownership action has an owner and next step.
- The final social profile inventory is recorded.
- ClickUp reflects the true status.

## Common Mistakes

- Adding social links without opening and verifying the profile.
- Linking to a personal profile instead of the business profile.
- Linking to a wrong-location franchise or old business page.
- Adding inactive profiles just to fill icons.
- Forgetting to check whether social profiles link back to the website.
- Mixing this process with social posting or GBP posting.
- Creating or claiming profiles without approved access.
- Leaving placeholder social links on the website.
