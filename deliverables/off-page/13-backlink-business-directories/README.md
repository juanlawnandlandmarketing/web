## Purpose

Process 13 covers business-directory backlink acquisition for SAE clients. The goal is to create or update legitimate directory listings that can provide a useful backlink, entity signal, or local relevance signal without creating spam, duplicate profiles, or inaccurate NAP data.

This process is related to citation work, but it is not the same thing.

- Citation building focuses on consistent business identity.
- Citation monitoring focuses on keeping that identity accurate.
- Business-directory backlink acquisition focuses on earning useful directory/profile links and documenting the result in the backlink tracking file.

The goal is simple:

- Find relevant business, local, industry, and green-industry directories.
- Use approved client business data and client-specific listing emails.
- Submit only legitimate listings that fit the client and service package.
- Avoid low-quality, paid-only, duplicate, or risky directories.
- Track each submission, live URL, login owner, verification status, and blocker.
- Keep completed links ready for backlink reporting and future monitoring.

## Target State

Every business-directory backlink run should produce a clear backlink acquisition log.

The ideal system:

1. Confirms the client is eligible for SAE backlink acquisition.
2. Loads the approved NAP and business profile data.
3. Uses the client-specific listing email when account creation is required.
4. Finds relevant business-directory opportunities.
5. Rejects spammy, duplicate, irrelevant, or risky directories.
6. Prepares submission-ready profile fields.
7. Assigns any login, email verification, payment, CAPTCHA, or ownership step to a human.
8. Records each submitted, live, pending, rejected, or blocked opportunity in the backlink tracking file.
9. Verifies live backlinks before marking the ClickUp work complete.

Automation should reduce mistakes and prep the work, not blindly create accounts or submit low-quality listings.

## Eligibility Rules

Before doing any submission work, confirm:

- The client is an SAE client.
- The service package includes backlink acquisition.
- The client has approved business identity data.
- The directory opportunity fits the client services and market.
- The listing will not create duplicate or conflicting NAP records.
- The backlink tracking file exists or can be updated.
- A client-specific listing email is available when account creation is needed.

Do not perform listing creation for Green Accelerator clients unless the service scope changes and Juan or Matt approves it.

## Directory Opportunity Types

Prioritize directories that create a real business signal.

| Directory Type | Examples | Decision |
|---|---|---|
| Local business directories | City, county, regional business indexes | Use when the market and category fit. |
| Green-industry directories | Lawn, landscape, hardscape, irrigation, nursery, contractor directories | Use when relevant and not spammy. |
| General business profiles | Legitimate business listing platforms | Use when the profile is trustworthy and NAP-safe. |
| Competitor-proven directories | Directories where local competitors have live listings | Review carefully, then use if relevant. |
| Chamber/community pages | Local chambers or business associations | Route to Process 16 if membership/payment is the main gate. |
| Social/profile platforms | Social/entity profiles | Route to Process 14 if the main work is social profile creation. |
| Spam/SEO directories | Thin, automated, irrelevant, paid-link, or suspicious sites | Reject. |

## Workflow

### 1. Confirm Scope and Eligibility

Start from `3. Backlink Acquisition`.

Confirm:

- Client is SAE.
- Backlink acquisition is included in the fulfillment scope.
- The requested run is monthly backlink work, not citation cleanup or reporting.
- A backlink tracking file exists.
- The client-specific listing email and access path are available from the ClickUp-controlled source.

Do not place client credential links or passwords in this SOP, GitHub, public notes, or dashboard text. Keep credential access inside the approved ClickUp/Google Sheet workflow.

### 2. Load the Approved Client Profile

Use the same source-of-truth discipline as citation work.

Collect:

- Business name.
- Address or service-area behavior.
- Phone.
- Website URL.
- GBP URL.
- Primary and secondary categories.
- Services.
- Service areas.
- Hours.
- Business description.
- Logo and approved photos.
- Social URLs.
- Existing listing email.

If the website, GBP, citation baseline, and client notes conflict, pause and resolve the source of truth before creating new directory listings.

### 3. Build Candidate Directory List

Find opportunities from:

- Competitor backlink analysis.
- Searches for `[service] directory [city/state]`.
- Searches for `[industry] business directory`.
- Green-industry association and contractor directories.
- Existing internal backlink files.
- Prior successful directory examples.
- Local business directories that fit the client market.

For each candidate, capture:

- Directory name.
- URL.
- Category or niche.
- Why it is relevant.
- Example competitor listing if applicable.
- Whether it allows a website link.
- Whether the link appears public.
- Account requirement.
- Verification requirement.
- Payment requirement.
- Duplicate risk.
- Initial decision: approve, review, reject, or blocked.

### 4. Score and Filter Opportunities

Use a simple quality gate.

| Factor | Good Sign | Reject/Review Sign |
|---|---|---|
| Relevance | Directory matches service, industry, or local market | Directory is unrelated or generic spam |
| Link value | Public profile includes website link | Link is hidden, nofollow-only is acceptable but still must be useful |
| Trust | Real business directory with visible listings | Thin SEO directory, scraped content, malware, or fake metrics |
| NAP safety | Allows accurate business identity | Forces wrong address/category or creates duplicate profile |
| Cost | Free or approved paid option | Payment required without approval |
| Access | Client email can be used cleanly | Requires personal/team account misuse |
| Duplication | No existing profile or existing profile can be corrected | Duplicate profile likely or old vendor controls it |

Reject any directory that would make the client look spammy or create entity confusion.

### 5. Check Existing Listings Before Creating New Ones

Before submission, search the directory for:

- Existing client listing.
- Old business name.
- Old phone.
- Old address.
- Old domain.
- Duplicate profiles.
- Claimed profiles controlled by someone else.

If a listing already exists, do not create a duplicate. Decide whether to:

- Log it as already live.
- Correct it.
- Claim it.
- Assign it for manual cleanup.
- Reject the opportunity for now.

### 6. Prepare the Submission Packet

For every approved opportunity, prepare:

- Directory URL.
- Client-specific listing email to use.
- Business name.
- Phone.
- Website URL.
- Address or service-area treatment.
- Category.
- Short description.
- Long description.
- Services.
- Hours.
- Logo/photo notes.
- Social URLs.
- Owner/contact field if required.
- Verification steps expected.
- Tracking row notes.

Descriptions should be clear and factual. Do not stuff keywords, invent awards, invent years in business, invent licenses, or claim services the client does not provide.

### 7. Submit or Assign Manual Steps

Classify each directory before action.

| Submission Type | Rule |
|---|---|
| Safe research/prep | Koga researches, scores, and prepares the listing packet. |
| Human submission | Human creates the listing because login, CAPTCHA, verification, or account risk is involved. |
| Assisted submission | Koga prepares fields and the human performs the final action. |
| Already live | Log the live URL and no new submission is needed. |
| Needs correction | Existing profile needs NAP/category/link cleanup. |
| Paid review | Payment or membership needs explicit approval. |
| Blocked | Missing access, client verification, directory issue, or unclear ownership. |

Use client-specific listing emails for listing creation when account creation is required. Do not use generic Gmail accounts or personal/team credentials unless the approved workflow explicitly allows it.

### 8. Verify Submission and Live Link

After submission or publication, verify:

- Listing exists under the correct business name.
- Website link is present.
- Website link points to the correct canonical URL.
- NAP is accurate.
- Category is reasonable.
- Description does not contain placeholders or AI leakage.
- Profile is live, pending, rejected, or blocked.
- Any email verification step has an owner.
- Any paid upgrade prompt was not accepted without approval.

If the listing is pending, log the pending status and follow-up date instead of marking it complete.

### 9. Update the Backlink Tracking File

Use the `Backlink file update` task after each batch.

Track:

- Client.
- Directory/platform.
- Opportunity URL.
- Submitted listing URL.
- Live profile URL.
- Target website URL.
- Submission date.
- Status: live, submitted, pending verification, rejected, blocked, duplicate, already live.
- Email/account used, without exposing passwords.
- Owner.
- Follow-up date.
- Notes and evidence.

Do not mark ClickUp complete until the tracking file reflects the real status.

### 10. Close Out ClickUp

Close the workflow only when:

- Approved opportunities were submitted or explicitly rejected.
- Live/pending/blocked statuses are documented.
- The backlink tracking file is updated.
- Manual blockers have owners.
- No duplicate or wrong-NAP listings were created.
- Any verification follow-up is scheduled.

## Completion Criteria

The process is complete when:

- The client scope is confirmed as SAE backlink acquisition.
- Directory opportunities are researched and scored.
- Approved submissions are completed or assigned to a human.
- Rejected and blocked opportunities are documented.
- Live links are verified where available.
- Pending verifications have owners and follow-up dates.
- The backlink tracking file is updated.
- ClickUp reflects the true status.

## Common Mistakes

- Treating spam directories as good backlinks.
- Creating duplicate profiles instead of correcting existing listings.
- Using generic Gmail accounts instead of approved client-specific listing emails.
- Performing listing creation for Green Accelerator clients.
- Accepting paid listings without approval.
- Forgetting to update the backlink tracking file.
- Marking the task complete while listings are still pending verification.
- Mixing citation cleanup, social profile creation, chamber memberships, and business-directory backlinks into one unclear task.
