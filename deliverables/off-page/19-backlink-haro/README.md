# Backlink - HARO / Connectively

**Category:** Off-Page SEO
**Automation Readiness Score:** 6/10 - Partial automation possible

## Purpose

Process 19 covers backlink opportunities from journalist-source request platforms such as HARO and Connectively. The goal is to win legitimate earned-media mentions or backlinks by responding quickly to relevant reporter, publisher, or expert-source requests where a Lawn & Land client can provide useful green-industry expertise.

This process is not generic link building.

- Business directory links belong to Process 13.
- Local press pitch outreach belongs to Process 18.
- Press release upload and Media Room workflows belong to Process 12.
- HARO / Connectively work starts from published source requests and only responds when the client is a real fit.

The goal is simple:

- Monitor approved source-request channels.
- Filter for lawn, landscape, outdoor living, home services, local business, seasonal maintenance, and small-business expertise requests.
- Reject weak, irrelevant, risky, or spammy opportunities.
- Draft short expert responses that answer the specific question.
- Get human/client approval when quotes, claims, personal attribution, or sensitive topics are involved.
- Submit before the deadline through the correct platform or sender.
- Track submissions, replies, placements, mentions, and backlinks.
- Verify live links and update the backlink tracking file.

## Target State

Every HARO / Connectively run should produce a clean response and placement log.

The ideal system:

1. Confirms which source-request channels are approved for the client or agency.
2. Monitors opportunities on the agreed cadence.
3. Scores requests by fit, deadline, publication quality, client expertise, risk, and backlink potential.
4. Drafts concise expert responses that answer the journalist's exact ask.
5. Gets approval before using a client's name, quote, credentials, customer story, project detail, or sensitive claim.
6. Submits through the approved platform, sender, or human owner before the deadline.
7. Tracks submitted, replied, accepted, published, rejected, blocked, and expired opportunities.
8. Verifies live placements, link status, brand mention accuracy, and tracking-file updates.

## Opportunity Types

Prioritize requests where the client can answer from real expertise.

| Opportunity Type | Examples | Decision |
|---|---|---|
| Green-industry advice | Lawn care, landscape design, hardscaping, drainage, irrigation, outdoor lighting | Strong fit when the client offers the service. |
| Seasonal home services | Spring cleanup, drought, storms, snow, leaf removal, pest prevention | Strong fit when timely and local/service-relevant. |
| Small-business/operator commentary | Hiring, route density, seasonal demand, pricing pressure, customer education | Use when the client can credibly comment. |
| Local expert input | City/region-specific property care or weather impacts | Use only when the client serves that market. |
| Product/tool roundups | Equipment, materials, software, customer communication tools | Use when the client has real experience. |
| Generic marketing or finance quotes | Broad business topics with weak green-industry fit | Usually reject unless client expertise is specific. |
| Pay-to-play or suspicious requests | Paid placement, guaranteed backlink, thin affiliate roundup | Reject. |

## Workflow

### 1. Confirm Scope and Access

Start from `3. Backlink Acquisition`.

Confirm:

- The client is eligible for backlink acquisition work.
- HARO / Connectively monitoring is in scope for the current run.
- The approved platform, inbox, or account access is available.
- Submission authority is clear.
- The backlink tracking file exists.
- No client credential, login, or password details will be placed in GitHub, the SOP, or public dashboard text.

If the platform, account, subscription, or inbox is not available, mark the run `blocked` and document the missing access.

### 2. Monitor Source Requests

Review the approved source-request channels.

Depending on the current account setup, this may include:

- HARO daily query emails.
- Connectively opportunity feed.
- Saved keyword alerts.
- Platform dashboards.
- Approved source-request digests.
- Internal forwarded opportunities.

Use keyword and category filters, but do not rely on keywords alone. A query can mention "home" or "outdoors" and still be a poor fit for a lawn and landscape client.

### 3. Score Opportunities

For each candidate request, score:

- Topic relevance.
- Client expertise fit.
- Geography fit.
- Publication quality.
- Deadline urgency.
- Response requirements.
- Risk or sensitivity.
- Need for client facts or quotes.
- Link/mention potential.
- Submission effort.

Use this decision scale:

| Decision | Meaning |
|---|---|
| Strong fit | Draft a response packet now. |
| Possible fit | Needs facts, approval, or better source match. |
| Weak fit | Do not spend time unless requested. |
| Reject | Irrelevant, risky, spammy, pay-to-play, or off-scope. |
| Blocked | Account, facts, quote, or approval unavailable. |

### 4. Prepare the Response Packet

For strong-fit requests, prepare:

- Request title/topic.
- Journalist or publication name if visible.
- Deadline and timezone.
- Query requirements.
- Recommended client/source.
- Draft quote or answer.
- Supporting fact or example.
- Approved bio/signature.
- Suggested website URL if the platform allows it.
- Risk notes.
- Approval needed.

Keep responses direct. Journalists do not need a blog post in their inbox.

### 5. Draft the Expert Response

A good response should:

- Answer the exact question.
- Lead with the useful answer, not the company pitch.
- Use plain, quotable language.
- Include one concrete example, detail, or field-tested observation.
- Stay within the query word limit.
- Avoid exact-match anchor text requests.
- Avoid generic AI phrasing.
- Include the approved source name, title, company, and website only where allowed.

Bad response pattern:

> "We are a leading provider and would love a backlink."

Good response pattern:

> "For homeowners, the biggest spring mistake is fertilizing before the soil is ready. If the lawn is still coming out of dormancy, the product works against the plant instead of helping it. Wait until the grass is actively growing, then match the application to the turf type and local weather."

### 6. Human/Client Approval

Human approval is required before:

- Submitting a quote under a client's name.
- Using a client's title, credentials, customer example, or project detail.
- Making claims about results, safety, pricing, demand, awards, licensing, or certifications.
- Responding to sensitive topics.
- Using a client photo, logo, or customer story.
- Creating or changing platform account information.
- Sending through a client-controlled email or profile.

Approval should define:

- Final response text.
- Source name and title.
- Approved facts.
- Approved link target.
- Submission owner.
- Deadline.
- Follow-up rules.

### 7. Submit or Assign Submission

Koga can prepare the final packet.

Humans submit when:

- Platform login is required.
- Client identity is used.
- The account belongs to a human or client.
- The opportunity requires judgment in form fields.
- The request involves sensitive claims.
- Juan or Matt has not explicitly authorized Koga to submit that exact response.

After submission, capture:

- Date/time submitted.
- Platform.
- Query/request title.
- Publication/journalist if visible.
- Source used.
- Submitted URL or response ID if available.
- Status.

### 8. Track Responses and Follow-Ups

Use consistent statuses:

| Status | Meaning |
|---|---|
| Candidate | Opportunity found, not reviewed. |
| Shortlisted | Looks relevant, needs draft or approval. |
| Drafted | Response packet prepared. |
| Approved | Ready for submission. |
| Submitted | Response sent through approved path. |
| Replied | Journalist or platform responded. |
| Accepted | Source selected or likely to be included. |
| Published | Placement is live. |
| Mention only | Client mentioned but no website link. |
| Rejected | Not selected or not suitable. |
| Expired | Deadline passed before submission. |
| Blocked | Missing access, approval, facts, or source. |

Do not mark a backlink as won until the published page is live and the link is verified.

### 9. Verify Placement and Backlink

When a placement appears, verify:

- Page is live and publicly accessible.
- Client/source name is correct.
- Company name is correct.
- Website link exists if claimed.
- Link points to the right URL.
- Link is not broken or redirected to the wrong page.
- Quote does not misrepresent the client.
- NAP, location, service, credential, or claim is not wrong.
- Page is appropriate for reporting.
- Backlink tracking file is updated.

If the publisher includes a mention without a link, track it as a mention. Do not inflate it into a backlink win.

## Completion Criteria

The process is complete when:

- Approved source-request channels were checked.
- Opportunities were shortlisted, rejected, expired, or blocked with reasons.
- Strong-fit response packets were drafted.
- Human/client approval was captured where needed.
- Approved responses were submitted or assigned before deadline.
- Submission and follow-up statuses were recorded.
- Published placements were verified.
- Links and mentions were classified accurately.
- Backlink tracking file was updated.
- ClickUp reflects the true status.

## Common Mistakes

- Treating HARO / Connectively like a generic backlink marketplace.
- Responding to every query instead of only real-fit opportunities.
- Sending long, generic, AI-sounding answers.
- Asking journalists for exact-match anchor text.
- Using unapproved client claims or quotes.
- Missing deadlines.
- Submitting through the wrong account.
- Marking a mention as a backlink when no link exists.
- Reporting a placement before the page is live.
- Reusing the Signal GeNeSYS, report, Looker Studio, or Imagify videos for this process.
